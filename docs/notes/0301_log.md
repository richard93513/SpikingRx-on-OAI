## 9) 下一步（必做）：UE 端要 dump LDPCdecoder() 之後的 llrProcBuf 才能對齊

### 9.1 你要達成的最終比對目標
當你把 UE 端真正的 decoder output dump 出來後，你應該得到：

- `cmp bitout_seg00.bin UE_ue_ldpc_out_seg00.bin => rc0=0`

只要 rc=0，就代表：
- 外部 decoder 與 UE decoder 完全一致（byte-by-byte）
- 之後你再比 cb_payload / tb_payload 才有意義

### 9.2 UE 端 patch 的原則（最小、最穩）
在 `LDPCdecoder(...)` 之後立刻 dump：

- dump 來源：`llrProcBuf`（或若 OAI 定義 output 其實是 rdata->c 之類也要確認，但依你貼的 code，output buffer 是 llrProcBuf）
- dump 時機：必須在 decode 之後
- 檔名：用你 grep 出來那個 `ue_ldpc_out_idx...` 格式
- out_len：仍用 `(BG==1)?(68*Z):(52*Z)` bytes

### 9.3 你完成 patch + 重新跑連線後，驗證流程
- 重新跑 UE/gNB 讓它產生新的 UE output dump（因為舊檔是 input dump）
- 在 raw 目錄重新跑外部 ldpctest_spx 產生新的 bitout_segXX.bin
- 再用 cmp 比對，期待 rc=0
