# SpikingRx × OAI UE Decoder 對齊 Debug 紀錄

## 1. 目標（Project Goal）

本工作目標：

讓 SpikingRx offline decoder 的輸入與 OAI UE LDPC decoder 的輸入完全一致。

也就是：

```text
SpikingRx pipeline input
==
UE LDPC decoder input (OAI)
```

具體來說：

```text
rm_exact_spx_segXX_i8.bin
==
rm_exact_segXX_i8.bin
```

這兩個檔案要 bit-exact identical。

---

## 2. 設計原則（非常重要）

本專案 debugging 的核心原則：

### 原則 1

不自己實作 PHY algorithm。

全部使用：

- `nr_deinterleaving_ldpc()`
- `nr_rate_matching_ldpc_rx()`
- `nr_dlsch_unscrambling()`

也就是直接調用：

- OAI 原始 function

原因：

- 3GPP implementation 非常容易 subtle mismatch
- OAI code 已經 verified
- 自寫版本容易出現 offset / sign / alignment error

### 原則 2

offline pipeline 必須與 UE pipeline 完全一致。

即：

```text
UE PHY pipeline
==
offline pipeline
```

---

## 5. 我們實際做過的檢查

以下是本次 session 已完成的檢查。

### 5.1 確認 LDPC config 完整一致

輸出：

```text
[CFG] BG=1
Zc=224
A=9480
C=2
K=4928
F=152
G=14400
Qm=2
```

這些與 UE 完全一致。

### 5.2 確認 segmentation

```text
C = 2
E = 7200 per segment
```

結果：

- seg0
- seg1

完全符合 UE segmentation。

### 5.3 確認 rate-unmatch 參數

```text
Ncb = 14784
Kprime = 4776
Foffset = 4328
```

UE公式：

```text
Foffset = K - F - 2Z
```

驗算：

```text
4928 - 152 - 448 = 4328
```

一致。

### 5.4 使用 OAI 原始 function

確認：

- `nr_rate_matching_ldpc_rx()`

已直接從 OAI 呼叫。

不是自寫。

### 5.5 對比結果

```text
cmp rm_exact_spx_seg00_i8.bin rm_exact_seg00_i8.bin
```

結果：

```text
451 364 14
452 364 14
453 13 365
```

特徵：

```text
365 ↔ 13
364 ↔ 14
```

這代表：

```text
sign flip
```

### 5.6 sign flip pattern 分析

例如：

```text
365 (octal) = -11
13  (octal) = 11
```

所以：

```text
LLR_spx = - LLR_UE
```

這是典型 scrambling mismatch pattern。

## 6. 找 UE 的 descrambling

我們 trace UE code。

搜尋：

- `nr_dlsch_unscrambling`

找到：

- `openair1/SCHED_NR_UE/phy_procedures_nr_ue.c`

關鍵程式：

```c
nr_dlsch_unscrambling(
    llr,
    G,
    0,
    dlsch_config->dlDataScramblingId,
    rnti
)
```

---

## 7. 確認 scrambling seed

我們 trace：

- `dlDataScramblingId`

來源：

- `openair2/LAYER2/NR_MAC_UE/nr_ue_procedures.c`

程式：

```c
if (pdsch_config->dataScramblingIdentityPDSCH)
    dlDataScramblingId = dataScramblingIdentityPDSCH
else
    dlDataScramblingId = physCellId
```

因此：

```text
scrambling seed = dlDataScramblingId
```

## 8. UE descrambling algorithm

UE 使用：

- `nr_dlsch_unscrambling()`

實際內部：

- `nr_codeword_unscrambling()`

檔案：

- `openair1/PHY/NR_TRANSPORT/nr_scrambling.c`

signature：

```c
void nr_codeword_unscrambling(
    int16_t *llr,
    uint32_t size,
    uint8_t q,
    uint32_t Nid,
    uint32_t n_RNTI
)
```
