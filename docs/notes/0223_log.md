## 5. 核心定位：你比較的是「bits(1bit/byte)」但 UE dump 的 tb 是「bytes packed」

你做了「把 decoded_bits（1bit/byte）pack 成 bytes」去比 UE TB bytes：

```bash
python3 - <<'PY'
import numpy as np
dec_bits = np.fromfile("decoded_bits_f0786_s11_from_dunmatch.bin", dtype=np.uint8)
ue_tb    = np.fromfile("f0786_s11_ue_tb_idx078611_rnti1234_dlsch00_harq12_round00_rv0.bin", dtype=np.uint8)

def pack(bits, bitorder):
    return np.packbits(bits.astype(np.uint8), bitorder=bitorder)

for bo in ["big","little"]:
    dec_bytes = pack(dec_bits, bo)
    mism = int((dec_bytes != ue_tb).sum())
    print(f"bitorder={bo}: byte_mismatch={mism} / {ue_tb.size}")

for bo in ["big","little"]:
    dec_bytes = pack(1-dec_bits, bo)
    mism = int((dec_bytes != ue_tb).sum())
    print(f"bitorder={bo} + invert: byte_mismatch={mism} / {ue_tb.size}")
PY
```

得到：
- byte mismatch 約 1178~1180 /1185（幾乎全錯）

意義：不是單純 bitorder big/little 或 invert 能解釋，表示：
- 你產生的 bytes 和 UE tb bytes 的 mapping 根本不同

可能原因：
- 你抽 bits 的 offset/方式不對
- decoder output 的 outMode 並不是「1bit/byte」直覺格式（可能 packed / 或不同 hard bit layout）
- TB 重組（filler/CB CRC/bit slicing）與 UE 端不一致

### Step C：用 python 自動找「你的 seg00 bytes」在 UE seg00 裡面的出現位置（精準定位 offset）

```bash

print("pattern_len=32 pos_in_ue =", pos)

```

- UE seg00 = 某種位移後的 payload bytes  
你現在的輸出只差一個「起點 offset」或「bit offset」。

### Step D：如果確定是 offset shift（循環位移），就直接修 pack_bits_msb_first 的 bit 起點

- 從 decoded_cb_bits 的 info_with_crc 開始，把 payload_bits pack 成 bytes

- 有 bit-level offset（例如從 bit k 開始）
- 或 decoded_cb_bits 本身的 bit 方向與 UE 相反（MSB/LSB）
