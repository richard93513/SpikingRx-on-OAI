## 3️⃣ 關鍵發現（最重要）

我們最後找到這段：

```text
rx_power_avg_dB = dB_fixed(...)
n0_power_avg_dB = dB_fixed(...)
```

👉

🔴 意義：

```text
OAI SNR =
(平均 + log + 量化)
```

👉 不是 raw SNR

---

### 👉 所以全部失敗的原因

你拿的全部量：

```text
都不是「raw SNR」
```

而是：

| 類型 | 問題 |
|------|------|
| nvar | 被 scale |
| H² | channel dominate |
| rxdataF | signal dominate |
| eqsymb | normalize |
| LLR | clipped |
| UE SNR | averaged + quantized |

### ✅ 最佳說法（可直接用）

```text
在 OAI 接收端，SNR 相關 measurement（如 rx_power_avg_dB − n0_power_avg_dB）經過平均與量化，無法提供足夠解析度來區分不同 AWGN 設定。
因此本研究改以 RFsim/channel model 中實際注入之 AWGN 作為 SNR 定義，於加噪點直接量測 signal 與 noise power，得到 ground-truth SNR。
由於 SpikingRx 與原始 OAI receiver 均在相同 channel 條件下測試，此 SNR 定義可公平比較兩者 BER 表現。
```
