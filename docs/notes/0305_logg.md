# 2. 各 dump 檔案意義

## rm_exact_segXX_i8.bin

LDPC decoder **真正輸入**

型態：

```
int8
```

長度：

```
out_len = (BG==1 ? 68*Zc : 52*Zc)
```

本例：

```
BG = 1
Zc = 224

out_len = 68 × 224
out_len = 15232 bytes
```

所以：

```
15232 bytes / segment
```

---

## ue_c_segXX.bin

UE decoder output codeblock payload

```
Kr_bytes = (K >> 3) − (F >> 3) − (C>1 ? 3 : 0)
```

本例：

```
594 bytes
```

---

## ue_tb.bin

UE 最終 TB payload

```
A / 8
```

本例：

```
A = 9480 bits
TB = 1185 bytes
```

# 5. 發現的重要問題

UE decoder output 是：

```
bytes path
```

不是：

```
bit extraction + pack
```

因此：

```
ldpctest_spx 必須：

memcpy(llrProcBuf → c[r])
```

而不是：

```
bit_off = 2Z
extract bits
pack
```

---

# 6. Buffer overflow bug

錯誤版本：

```
K_bytes = 616
```

直接當 decoder output buffer

導致：

```
malloc(): corrupted top size
```

原因：

```
OAI decoder 寫超過 616 bytes
```

---

# 7. 正確解法

使用 OAI 相同 buffer：

```
27000 bytes
```

例如：

```
llrProcBuf = memalign(32, 27000)
```

然後：

```
memcpy(c_bytes[r], llrProcBuf, K_bytes)
```

# 13. TB mismatch 原因

```
TB CRC = 24 bits
= 3 bytes
```

本例：

```
594 + 594 = 1188
A/8 = 1185
```

差：

```
3 bytes
```

因此 TB assembly 必須：

```
只 copy A/8 bytes
```
