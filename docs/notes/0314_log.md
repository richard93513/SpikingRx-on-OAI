# 9. 發現 Readout Bottleneck

原始 readout：

```
AdaptiveAvgPool2d(4×4)
```

如果 input 是：

```
32×64
```

pooling 後：

```
4×4
```

問題：

```
64 columns → 4 columns
```

頻域資訊幾乎全部被壓掉。

---

# 10. Readout 修改

修改：

```
pool_hw = (4,8)
```

變成：

```
32×64 → 4×8
```

保留更多頻域資訊。

# 14. SpikingRx 訓練問題真正根因分析

這一段是最重要的分析。

---

# 根因 1：Input Encoding 破壞 constellation geometry

目前 pipeline：

```
1272 subcarriers
↓
mean pooling
↓
64 columns
```

問題：

```
mean pooling 會破壞 constellation information
```

例如：

QPSK constellation：

```
(+1,+1)
(+1,-1)
(-1,+1)
(-1,-1)
```

但如果 pooling：

```
平均後
symbol geometry 消失
```

導致：

```
demapper mapping 無法學習
```

---

# 根因 2：Demapper 本質是幾何問題

LLR 計算公式：

```
LLR = log(P(bit=0)/P(bit=1))
```

對於 QPSK：

```
LLR ≈ distance to constellation boundary
```

本質是：

```
complex plane geometry
```

而 CNN + pooling 不一定能學到這種幾何關係。

---

# 根因 3：Resource Grid 與 LLR mapping 並非 local

LLR 不只依賴：

```
單一 subcarrier
```

還依賴：

```
equalization
channel estimation
noise variance
```

但目前模型 input：

```
只有 fullgrid
```

沒有：

```
channel estimate
noise power
```

因此 mapping 本身可能是不完整的。

---

# 15. 為什麼模型會卡在 sign≈0.5

如果 mapping 不存在或極難學：

模型最安全策略：

```
輸出接近 0
```

因此：

```
sign accuracy ≈ 0.5
```

這正是目前看到的結果。

```

```


## (2) 增加頻域解析度

目前：

```
32×64
```

可以嘗試：

```
32×128
```

保留更多 subcarrier。

---

## (3) 視覺化 fullgrid vs LLR

畫：

```
symbol scatter plot
LLR distribution
```

確認 mapping 是否可學。

# 17. 什麼情況下再增加 epoch

如果未來實驗出現：

```
val_norm_mse < 0.95
sign accuracy > 0.6
```

代表模型開始學到 mapping。

此時才值得：

```
max_epoch = 50~100
```

否則：

```
只是浪費時間 overfit
```
