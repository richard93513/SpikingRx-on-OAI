## 25. 為什麼小資料量可以、大資料量不行？

這裡我們最後確定的理解是：

### 小資料量（1 / 8 bundles）

模型其實只是在：

- memorize mapping

也就是：

```text
x1 → y1
x2 → y2
...
```

並不代表它學到了 `fullgrid→LLR` 的通用規律。

### 大資料量（1803 bundles）

這時 sample 間：

- channel 不同
- noise 不同
- HARQ / RV 可能不同
- 其他 implicit 條件也不同

所以真實 mapping 其實不是單純：

```text
grid → LLR
```

而比較像：

```text
(grid, channel, noise, rv, ...) → LLR
```

但目前 supervision 只有最終：

- `LLR loss`

所以模型要自己從 DMRS 去內隱學會：

- DMRS → channel estimation
- channel → equalization
- equalization → demapping

這條鏈太長，梯度太遠，對現在資料與模型規模來說太難。

## 26. 我們的研究方向最後怎麼定？

你特別強調這點，我們最後也已經講清楚：

不能走 `equalized grid → LLR` 當最終方向

因為那樣只是在學 demapper，不符合你現在要做的：

- 取代 channel estimation + equalization + demapper

所以最後的方向是：

- 不是改成 inference 用 equalized grid
- 而是：training 時加入中間 supervision
- inference 時仍然 `fullgrid → LLR`

## 27. 最終確定的下一步：三個 loss 的訓練

我們最後決定的方向是：

### network 流程

```text
fullgrid
   ↓
shared encoder
   ↓
channel head
   ↓
equalized symbol head
   ↓
LLR head
```

### training loss

```text
L_total = L_llr + α L_eq + β L_channel
```

其中：

#### 1. `L_channel`

讓 network 學：

```text
DMRS → channel
```

可以用：

```text
MSE(H_pred, H_est)
```

#### 2. `L_eq`

讓 network 學：

```text
channel → equalization
```

可以用：

```text
MSE(eq_pred, eq_true)
```

#### 3. `L_llr`

最後仍然保持：

```text
BCE(LLR_pred, LLR_true)
```

---

## 28. 這樣做的意義

這樣 inference 仍然是：

```text
fullgrid → LLR
```

所以還是符合你要的：

- 直接從 fullgrid 學到 LLR

只是在 training 時，不再只給最遠的 LLR supervision，而是加上中間 supervision，讓模型比較容易學到：

- DMRS → channel
- channel → equalization
- equalization → demapper
